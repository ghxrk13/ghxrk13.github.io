apld-edge

Personal algorithmic trading platform — sqlite-backed market-data daemon, Claude-driven signal scoring, append-only decision log. End-to-end, deployed.

Status: live Python · Claude · SQLite · AWS · Async

What it is

A single-ticker research and trading platform built around a continuous 24/7 data-collection daemon and a Claude-driven advice engine. The daemon enriches a local sqlite store with price, implied volatility, news, SEC filings, peer prices, and broader market signals. A periodic brief step ships the relevant state to Claude, which produces a structured recommendation logged to an append-only decision file.

Design constraints that shaped it

  • Statelessness of the LLM. Claude has no memory between runs, so the whole platform is built around feeding the right context every time and treating the decision log as the durable surface.
  • Resilience over cleverness. Collection daemon must survive market hours, network outages, and IDE restarts. Built with cron + service-style supervision.
  • Auditable decisions. Every recommendation Claude produces is logged with the exact input snapshot, so the rationale can be replayed and graded against actual outcomes.

What it does well

  • End-to-end data pipeline: collection → enrichment → briefing → recommendation → logging.
  • Multi-environment deploy: works on local Surface Book + AWS t4g.nano + RunPod GPU bursts.
  • Secrets via Bitwarden CLI — works on Windows-SSH and Linux equally; no env-file leaks.

What it isn't

Not a fund. Not financial advice. Personal research tool, sized for personal capital and personal risk tolerance.
