Status: shipped (and ongoing) Django · Python · LLMs in production · AppSec

Context

DefectDojo is the dominant open-source platform for managing application security findings — vulnerability triage, deduplication, lifecycle, integrations across scanners. A team I work with maintains a fork tailored to high-rigor deployments. My contributions sit at the intersection of the application backend, developer experience, and the LLM-augmentation surface.

Features shipped

  • Multi-SCM abstraction layer. A pluggable helper that generates correct deep-links from a finding to the offending code, regardless of whether the source lives on GitHub, Gitea, Bitbucket, or Azure DevOps. Replaces hard-coded GitHub-only handling in several call sites.
  • VS Code integration. One-click vscode:// handlers from findings → opens the file at the right line in the reviewer's editor, including in air-gapped / code-server deployments.
  • Governance-aware infrastructure. Tooling for filter-branch / commit-history anonymization so the fork can be mirrored cleanly between collaborator accounts without leaking contributor metadata that doesn't belong in the public surface.
  • LLM-augmented triage (in development). Claude-driven summarization and clustering of findings to reduce reviewer fatigue on noisy scan output.
